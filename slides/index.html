このセッションのテーマはお手軽さの追求です

nowというサーバーにもクライアントにも同時に存在するかのように振る舞うオブジェクトを提供する

// On the server
// Sets the value of every client's `now.x` to 'test'
everyone.now.x = 'test';

// Calls a function named `test` on every client, ignoring clients that do not have `now.test`
everyone.now.test();

特徴

  • 変数が共有されてる感じでコーディングできる
  • 勝手にフォールバックしてくれる(後ろでSocket.ioが動いている)
  • websoketのチマチマしたコードを書かなくていい
  • 詳しく知りたければ徹底解説 HTML5 APIガイドブック コミュニケーション系API編を読むといいです

CoffeeScript

(私版)CoffeeScriptの何がいいの?

  • function が ->
  • bindが=>
  • 中括弧不要(インデント)
  • クラスの宣言が便利
  • thisを@
  • 文字列埋め込み楽ちん "変数は#{var}"
  • リスト内包表記が使える
  • for in, for ofループが楽
  • ifの後置(perlみたい)

(注意) CoffeeScript

  • javascriptにコンパイルするだけなので結局javascriptがわからないとデバッグできない
  • 他言語(Ruby,Python)を知らないと学習コストが無駄にかかる
  • 自分に合っているかどうかは各自で判断してください
